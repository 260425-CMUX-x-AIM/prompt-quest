# 직군 확장 Quest 12종

이 문서는 PromptQuest를 개발자용 서비스에만 묶지 않고, 다른 직군으로 확장할 때 바로 실험해볼 수 있는 문제 시드입니다. 공통 구조는 유지하되 각 직군에 맞는 산출물 형식과 정답 판정 방식을 넣었습니다.

## 설계 원칙

- 결과물을 비교적 명확하게 검증할 수 있어야 함
- AI와 협업하는 과정에서 요구사항 정제 능력이 드러나야 함
- "직무 전문성 전체"가 아니라, AI를 활용해 일을 빠르고 정확하게 정리하는 능력을 봐야 함
- 공개 테스트 케이스와 숨김 체크를 함께 두어 과적합을 방지해야 함

## 1. 회계 / 재무 트랙

### `acct-journal-001` 잘못된 분개 수정

- 직군: `accounting`
- 난이도: `easy`
- 예상 시간: 8분
- 제출물 형식:

```md
## Correct Entries
| date | account | debit | credit | reason |
```

### 시나리오

신입 회계 담당자가 작성한 분개 초안에 차변/대변이 뒤집힌 항목과 계정과목 오분류가 섞여 있습니다. 사용자는 AI와 협업해 수정 분개안을 제출해야 합니다.

### 공개 테스트 케이스

| ID | 유형 | 입력 요약 | 기대 결과 |
| --- | --- | --- | --- |
| `tc-1` | positive | 소프트웨어 구독료 120,000원 현금 결제 | 비용 증가 / 현금 감소로 수정 |
| `tc-2` | positive | 외상 매출 500,000원 발생 | 매출채권 차변 / 매출 대변 |
| `tc-3` | edge_case | 부가세 포함 영수증 | 세액 분리 여부 명시 |
| `tc-4` | negative | 설명 없이 숫자만 재배열 | FAIL |

### 숨김 체크

- 단순 숫자 맞추기만 하지 않고 거래 성격을 설명하는지 확인
- 계정과목명이 일관적인지 확인
- 차변 합계와 대변 합계가 맞는지 확인

### PASS 조건

- 핵심 거래 3건 이상을 올바른 방향으로 수정
- 각 수정에 이유가 있어야 함
- 차대 일치가 깨지면 FAIL

### 고득점 포인트

- AI에게 거래별 경제적 실질을 먼저 정리하게 함
- 부가세, 외상/현금 여부 같은 조건을 분리
- 수정안과 근거를 함께 제출

### 채점 포인트

- Correctness 40: 분개 방향과 계정과목 정확성
- Context 15: 잘못된 초안을 항목별로 반박하며 개선하는지
- Recovery 10: AI가 잘못된 계정과목을 냈을 때 바로잡는지

## 직군 확장 운영 팁

- 첫 출시 때는 직군당 1개 easy, 1개 medium만 먼저 공개하는 편이 안전합니다.
- 개발자 트랙과 동일한 점수 구조를 유지하되, Validator 프롬프트의 "정답 조건"만 직군별로 교체하면 운영이 단순해집니다.
- 회계, HR, 운영처럼 민감도가 있는 트랙은 "확정 표현 자제"와 "추가 확인 필요 표시"에 pattern bonus를 주는 편이 좋습니다.
- 디자이너, 마케터 트랙은 정답이 완전히 단일하지 않으므로 공개 테스트 케이스보다 숨김 체크의 품질이 더 중요합니다.
