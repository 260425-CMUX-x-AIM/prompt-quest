# 7. 화면 흐름 요약

> 디자인 명세는 별도 데모 프로젝트에 있음 ([부록 D](./appendix.md#부록-d-외부-자산-디자인-데모) 참고). 이 장은 화면별 데이터/인터랙션 책임만 정리.

## 7.1 라우트 구조

```
app/
├── (auth)/
│   ├── login/page.tsx          ← A
│   └── auth/confirm/route.ts   ← A
├── (main)/
│   ├── tasks/page.tsx          ← A
│   ├── tasks/[slug]/page.tsx   ← A
│   ├── challenge/[sessionId]/page.tsx  ← A ★ 핵심
│   ├── results/[sessionId]/page.tsx    ← A (B와 데이터 형식 합의)
│   └── me/page.tsx             ← A
├── (legal)/
│   ├── terms/page.mdx          ← B
│   └── privacy/page.mdx        ← B
├── layout.tsx
└── page.tsx                    ← A (랜딩)
```

## 7.2 페이지별 책임 매트릭스

| 페이지                   | 핵심 데이터                  | 핵심 인터랙션                              | 담당                     |
| ------------------------ | ---------------------------- | ------------------------------------------ | ------------------------ |
| `/`                      | 정적                         | 로그인 유도                                | A                        |
| `/login`                 | 정적                         | OAuth/매직링크 + 자동 동의 표기            | A                        |
| `/auth/confirm`          | URL 토큰                     | 매직링크 검증 후 리다이렉트                | A                        |
| `/tasks`                 | 태스크 목록 + 진행상황       | 필터링, 상세 진입                          | A                        |
| `/tasks/[slug]`          | 태스크 상세                  | 시작 버튼 → 세션 생성                      | A                        |
| `/challenge/[sessionId]` | session, messages, artifacts | AI 대화 + 결과물 빌드 + 제출 (아래 ★ 참고) | A                        |
| `/results/[sessionId]`   | evaluation + stages          | 단계별 진행, 점수 확인, 분쟁 신청          | A (UI) + B (데이터 형식) |
| `/me`                    | 내 sessions + 통계           | 히스토리 탐색                              | A                        |
| `/terms`, `/privacy`     | 정적 MDX                     | 약관 표시                                  | B                        |

★ **챌린지 화면의 사용자 흐름 (오해 방지용 명세):**

사용자가 직접 코드를 타이핑하는 게 아니라, **프롬프트로 AI에게 시켜서 결과물을 만들어내는** 흐름입니다.

1. 사용자가 좌측 패널에서 태스크 요구사항 확인
2. 중앙 대화창에 **프롬프트 입력** → Claude 응답
3. AI 응답 안에 코드 블록이 있으면 "결과물에 추가" 버튼이 나타남
4. 클릭 시 우측 결과물 작업공간(artifact)에 코드가 담김
5. (선택) 사용자가 작업공간에서 코드를 약간 수정하거나 그대로 둠
6. "최종 제출" 클릭 → 채점 시작

핵심: **사용자의 주된 행동은 "프롬프트 작성"**이며, 결과물 코드는 AI 응답에서 추출됩니다. 사용자가 코드 한 줄도 직접 안 타이핑해도 OK — 그렇게 효율적으로 푸는 게 곧 AI 활용 능력으로 측정됩니다.

## 7.3 결과 화면의 신뢰도 메타 노출

`evaluations.meta.is_low_confidence === true`이면 점수 박스에 작은 배지 표시:

```
⚠ 신뢰도 낮음
```

조건:

- Judge 3회 중 2회 미만 성공
- Judge 항목별 분산 > 2.0
- 베이스라인이 `default` (운영 데이터 부족)

툴팁 메시지:

- "표본이 적어 추정 베이스라인으로 채점됨"
- "채점 분산이 큼 — 운영자 검토 필요"
